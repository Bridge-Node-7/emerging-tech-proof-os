# Contributing

Thank you for helping improve Emerging Tech Proof OS.

## How to Contribute

1. Read `DOCTRINE.md` and `LIMITATIONS.md` first.
2. Open issues or pull requests for improvements to workflows, templates, or documentation.
3. All evidence-related contributions must follow the evidence card format and pass the automated checks.
4. AI-assisted contributions must complete the AI Governance Gate.
5. Completed evidence cards should go in `docs/evidence-cards/`.
6. Completed decision-ready briefs should go in `docs/briefs/`.
7. Completed governance gates or review notes should go in `docs/governance/`.

## Versioning

- `VERSION` should contain only the current semantic version, such as `v0.1.2`.
- Update `VERSION` and `CHANGELOG.md` before tagging a release.
- Keep release notes accurate. Do not describe sample or unreviewed artifacts as validated.

## Code of Conduct

- Evidence over hype.
- Human judgment is final on high-impact content.
- Synthetic or sanitized data only in public examples.
- Clear confidence scoring and limitations.

## Development

- Test workflow changes locally where possible.
- Update `CHANGELOG.md` with every significant change.
- Keep workflows minimal and well-commented.
- Prefer plain language that helps both people and AI agents understand the repository.
