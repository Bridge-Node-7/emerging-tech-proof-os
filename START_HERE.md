# Start Here

Emerging Tech Proof OS is a GitHub repository template for evidence-based emerging technology work.

It is designed to help people and AI agents organize a project without guessing what the repository is for.

## What This Is

A public foundation skeleton for:

- evidence cards
- governance checks
- AI-assisted content review
- basic repository quality checks
- validated releases

## What This Is Not

It is not:

- a standalone app
- a software package to install
- a complete production system
- a substitute for expert review
- a source of truth without validation

## Quick Start

1. Read `MISSION.md`.
2. Read `DOCTRINE.md`.
3. Use `templates/evidence-card.md` for claims.
4. Use `templates/ai-governance-gate.md` for AI-assisted content.
5. Use `templates/decision-ready-brief.md` when a decision needs a clear summary.
6. Save completed evidence cards to `docs/evidence-cards/`.
7. Save completed decision-ready briefs to `docs/briefs/`.
8. Save completed governance gates or review notes to `docs/governance/`.
9. Let the GitHub Actions checks run.
10. Review and validate before publishing.

## Which Template Do I Use?

| Template | Use it when |
|---|---|
| `templates/evidence-card.md` | A claim may influence a decision, public release, roadmap, or recommendation. |
| `templates/ai-governance-gate.md` | AI drafted, summarized, classified, scored, or organized any meaningful content. |
| `templates/decision-ready-brief.md` | A decision needs a structured summary with evidence, risks, confidence, and next steps. |

## Agent Instructions

If you are an AI agent using this repository:

- Do not invent evidence.
- Do not treat claims as validated unless the evidence card says they are validated.
- Keep observed facts separate from analysis or inference.
- Preserve confidence levels, limitations, and validation requirements.
- Do not add sensitive data, credentials, internal hostnames, personal data, or controlled information.
- Prefer small, clear edits over broad rewrites.
- Do not claim a release is validated unless human review status supports that claim.

## Main Path

```text
Claim → Evidence card → Governance check → Human review → Validated release
```
